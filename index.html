DISABILITY SA

  • Project Summary: The website isn’t helping to break the cycle of disability prejudice and isolation, by low participation and donor contribution.
  • Our Goal: Improve the website user experience so that it empowers unity and promotes participation with volunteers and donors
  • Duration: 3 weeks.
  • My Role: UX Designer/Researcher.
  • Team: 4 members
  • Tools: Figma, Google Drive, InVision and Miro.
    • Discovery UI Wireframes (RWD) Usability Testing Lessons Learned

    User Research

    For this project, testing was performed via remotely and in-person in order to gather feedback and visualize users' interaction with the website. We designed a testing plan based on the problem we focused on solving which was to improve the website's navigation and usability. Five participants were asked to provide their initial views of the DisabilitySA’s website, as well as their likes/dislikes & the accessibility. During these sessions, users were observed to see what they gravitated to first and where they clicked on the page. The repeated search choices on the page gave different search results per topic, causing the user even more confusion. Once feedback was recorded and analyzed from both prototypes, iterations were done as needed in order to meet users preferences and usability standards. We discovered users felt overwhelmed from the amount of text and buttons available from the homepage and further screens.
  • Optimized navigation
  • Donate button in header
  • Decreased text
  • Adaptive content
  • Progress bar
  • Broad donation plan to inspire contributions
  • Simplified donation page
  • Kept the use of action buttons for final step
  • Task 1 - Sign up to be a volunteer (100% completion rate)
  • Task 2 - Sign up for an event (100% completion rate)
  • Task 3 - Make a donation (60% completion rate)
  • Clear navigation for users to donate and participation
  • Continue to create accessibility for users
  • How to create an empathic experience for users
